"Search directories" panel : directory list and mouse click

Very difficult to write a proper title for this issue...

When in the "Search directories" panel of "Project build options", if one clicks on one of the directories that are listed (in any of the sub panels : "Compiler", "Linker" or "Resources"), then clicks on another one, the previous line doesn't get deselected. One then ends up with two (or more) lines selected at the same time.

This is typically the behavior one would expect if the shift key was pressed all along, yet it's not.

I'm using the latest nightly, but the problem appeared between SVN 7550 (worked fine) and 7917 (problem started). Unfortunately, due to the lack of nightly builds between those two versions, I am unable to provide a more precise date.

I also develop on linux, and this problem doesn't occur there.
