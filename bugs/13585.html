Reload dialog shows when autosave if enabled

When I have autosave enabled (automatically save source files is set to 1) and I change some file and wait 1 minute the "Reload file" dialog pops up telling me that the file has changed on disk. If I disable the autosave plugin it behaves correctly.

It is not a new bug. I know about it for very long time, but just today I understood the reason.

It happens on gentoo ~amd64 using codeblocks build from svn.

It happens on archlinux 32bit using the official release.
