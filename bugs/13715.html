Code Completion toolbar init issue

This is possibly related to a similar bug I found:

[ Bug #13299 ] CodeCompletion Toolbar is empty?

I am running Code::Blocks SVN 5020 with the Code Completion 0.7 plugin on Debian.

When I load the .cbp file of a project I'm working on, the Code Completion toolbar is blank and unusable when viewing my source files. This problem only occurs when the following conditions are met:

- When I load a .cbp file from the command line ("codeblocks project.cbp")

- When the .cbp file opens source file tabs that were open at last save.

This problem DOES NOT happen:

- If you open codeblocks first and THEN load a .cbp file from the menu.

- OR after you modify and save a file.

- OR if you close the source file tab and reopen it from the manager view.

I've filed this under Application::Error instead of Plugin::CodeCompletion because the error happens only when you load Code::Blocks a certain way.
