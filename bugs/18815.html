Perspectives become corrupt

I have had several incidents of the perspective becoming corrupt, with toolbars having locations outside the viewable screen. This has happened in every version I have used, both release and nightly builds for several years. It may be triggered by switching between single and multiple monitors but I can not be sure. The result is that I can not see the toolbars and the only solutions are to delete the perspective or edit the default.conf file manually. Sometimes editing the file is the only option because all other perspectives are similarly corrupt so one can not create a valid perspective.

A possible (quick) work-around would be to add a menu option to create a new default perspective with valid / sensible values for each element.
