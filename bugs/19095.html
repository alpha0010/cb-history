Incorrect display when watching structs

Watching a struct containing (typedef) structs with C::B 12.11 results into the structs being displayed as children of the previous variable.

In the current nightly (06 August 2013 build (9246)) this is only the case for the first struct, all following structs and variables are displayed correct.

An older build (svn build rev 8024 (2012/06/02 14:55:25) gcc 4.5.2 Windows/unicode - 32 bit) displays everything correct.

Sample project and screenshots are uploaded at

http://dasjojo.de/WatchesBug.zip
