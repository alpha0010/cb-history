nickwiz_berli0s

Uninstall does not remove default.conf

Win XP SP2, bin install v 8.02

(codeblocks-8.02mingw-setup.exe)

Uninstall does not remove default.conf.

Unwanted result are upon re-install old default.conf are used.

-- Example of problem ---

1. Had Bloodshed installed on XP box.

2. Installed Code::Blocks.

3. Code Blocks reffer to Bloodshed's MinGW.

4. Uninstalled Bloodshed

5. Un-Installed Code Blocks,

6. Installed Code Blocks,

- Now Code Blocks still reffer to MinGW of Bloodshed.

---

Trying to fix by changing path / and or / each executable:

Settings -> Compiler and debugger... -> Toolchain executables.

* but this did not work.

-- Fixed by:

1. Uninstall Code Blocks

2. Delete codeblocks folder from C:\Documents and Settings\xxxuserxxx\Application Data\

3. Installing Code Blocks
