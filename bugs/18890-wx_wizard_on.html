wx wizard on shared debug and static release targets

When I create a project as a dynamic link debug target, and then add another static link release target, the release target doesn't build.

Look into the project options, I found:

1. The "WXUSINGDLL" define for dynamic link is put in project level. Which will fail the build of newly added static link release target.

2. Win32 api libs(libkernel32.a, libuser32.a, libgdi32.a, ...) are missing. If I create the project as static link in the first place, I can see them been set in the project level. I think that is the problem?

CB version: 12.11
