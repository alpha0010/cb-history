Code::Blocks crashes with an X error when oxygen-gtk is used

Code::Blocks 10.05 does not start when oxygen-gtk is used as GTK+ theme. It just fails with the following error message after I selected my default compiler:

The program 'codeblocks' received an X Window System error.

This probably reflects a bug in the program.

The error was 'BadWindow (invalid Window parameter)'.

(Details: serial 12215 error_code 3 request_code 152 minor_code 3)

Changing the GTK+ theme fixes the the problem. All other wxgtk applications seem to work fine with oxygen-gtk.

I am running Archlinux with wxgtk 2.8.12.1, gtk+ 2.24.10 and oxygen-gtk 1.2.2.
