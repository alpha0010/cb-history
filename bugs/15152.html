%APPDATA% pointing to network folder in Windows

Code blocks version 8.02

Microsoft Windows XP SP 3

When APPDATA references a network shared folder (e.g. "\\somwhere\Application Data", Code::Blocks behaves incorrectly.

The "Application Data" folder has the read only attribute set, (However I can create files/folders manually through the explorer).

1. it fails to store the default configuration at exit.

- work around is to create codeblocks\share\codeblocks\scripts in the APPDATA folder.

2. Can't create project templates

- partial work around is to create codeblocks\UserTemplates. This however doesn't work fully.

I suspect code::blocks fails when trying to create folders.
