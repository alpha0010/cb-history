Fold/Unfold all glitch when only parts of code are affected

When using the Unfold All action, available through both Edit -> Folding and the right-click menu:

When only some (not all) blocks are unfolded, the editor will fail to update those blocks' visual effects (the "+" icon in the left margin and the line right beneath the opening symbol of the block ("{")), forcing the user to manually unfold them each once in order to get rid of the effect.

Using "Unfold all" twice will *not* take care of these glithces. The glitch is also persistant if you fold all on a file with already partially folded blocks and then unfold them again.

Be aware that it is only the two mentioned visual effects and *not* the contents of the blocks themselves that are not updated. The contents are folded and unfolded succesfully in all cases.
