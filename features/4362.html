Feature #4362 2008-11-04 00:32

ecormier

Build messages search for file on click

Some compilers output only a filename without a path for errors and warnings. For example:

"myfile.c", line 123, error: missing semicolon

When the file containing the error/warning is under a subdirectory of the project directory, clicking on the error message will not take you to the corresponding file/line. It would be nice if subdirectories could be searched for that file.
