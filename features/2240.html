Feature #2240 2006-05-29 15:55

mferit

autonaming workspace and project contain

I am pretty sure this falls under the category of "feature requeset" rather than "bug". I am using C::B under WinXP pro.

When I create a new project, the name of the project in the workspace browser reflects the kind of project I created (console project, etc.). It would be nice if the name were automatically set to the name I assigned to the project when I created it. I know I can change the name via the properties window, but that's a bit of a pain.

Also, when I create a new workspace, its name remains "Default Workspace" until again I change it manually. I would prefer that this too be automatic.

BTW, C::B is an awesome project. Keep up the good work! I am looking for an IDE to use in university 1st year programming courses and C::B is *almost* it. With the changes above, it would definitely be my first choice.
