Standardised paths in project files

Would it be possible to add an option such that project files consistently use the same relative path formats (such as forward slashes only).

We have recently updated our project so that it can be compiled on either Linux or Windows, but whenever a change to the project file is made on either platform, the relative paths all switch between backward and forward slashes.

Obviously this is showing up as a major change in source control each time, so it would nice if the project files used a standard format for paths.
