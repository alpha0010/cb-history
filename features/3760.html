sersh

Extended keyboard shortcuts

Currently keyboard shortcuts can be assigned to any item in the main menu. However, context menu has several items not available in the main menu - find declaration, find implementation, open include. It would be useful to be able to assign shortcuts to them.

Additionally, one key can only be assigned to one action. This resolves ambiguities, but in some cases it would be comfortable to reuse keys. For example, one key could be used for start and continue (debug), since only one of these can be available at any given time. First enabled item out of all assigned would be fine, but context-sensitive choosing could be even better (e.g. different actions assigned for same keys - one during editing and one during debugging).
