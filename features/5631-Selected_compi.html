Feature #5631 2013-03-22 17:14:00

pir-uk

Selected compiler list too cluttered!

C:B 12.11 (under Linux at least) presents a long list of 12 compliers and cross-compilers which add annoying clutter to the interface. These 'built-in' compilers (Intel C/C++, SDCC, LLVM, etc) cannot be deleted and I suspect will never be used by the vast majority of users. Is it possible to remove (most of) them in order to tidy-up the interface? Anybody wanting to add a new compiler or cross-compiler can easily do so. I suggest only the GCC compiler should be 'built-in' - users can add additional (cross-) compilers to suit their usage.
